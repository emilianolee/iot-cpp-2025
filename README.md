## 1일차
- 입출력 방식 : [C++](./dia%2001/cout.cpp)
    - cout
        - 출력을 담당하는 객체
        - std::cout << 을 이용하여 출력함
        - 출력을 위해서는 다음의 형태와 같음
            - std::cout << '출력대상';
    - endl
        - << std::endl 을 이용하여 개행(줄바꿈)함
        - 출력과 개행을 위해서는 다음의 형태와 같음
            - std::cout << '출력대상1' << '출력대상2' << '출력대상3' << std::endl;
    - cin : [C++](./dia%2001/cin.cpp)
        - 입력을 담당하는 객체
        - C언어에서의 scanf와 같음

- 함수 오버로딩 : [C++](./dia%2001/overLoading.cpp)
    - 함수의 다중정의를 의미함
    - C언어에서와는 달리 C++에서는 매개변수의 선언 형태(타입, 갯수)가 다르다면 동일한 이름의 함수정의를 허용함

- 매개변수의 디폴트 값 : [C++](./dia%2001/default.cpp)
    - 매개변수에 디폴트 값이 설정되어 있으면, 선언된 매개변수의 수보다 적은 수의 인자전달이 가능함
        - 디폴트 매개변수는 입력이 없을 때 적용됨
    - 전달되는 인자는 왼쪽에서부터 채워져 나가고, 부족분은 디폴트 값으로 채워짐
        - 즉, 디폴트 매개변수를 여러개 작성 시, 제일 오른쪽부터 순서대로 작성해야함

- 이름공간(namespace) : [C++](./dia%2001/namesp.cpp)
    - std를 활용한 객체를 불러오면 std를 안써도 됨. 형태는 다음과 같음
    - using namespace std; 을 사용하면
    - cout << '출력대상' << endl; 만 사용하면 됨

- 동적메모리 할당과 해제(new - delete) : [C++](./dia%2001/new.cpp)
    - new는 생성자를 호출함. 즉, 객체를 생성시킴
    - new는 생성자 호출이되므로 초기화가 가능함.
        - 생성자는 객체를 생성하고, 초기화시키는 기능을 가지는 특별한 메서드
    - 형태는 다음과 같다
        - 자료형 포인터 = new 자료형(크기)

- 참조자(Reference) : [C++](./dia%2001/reference.cpp)
    - 자신이 참조하는 변수를 대신할 수 있는 또 하나의 이름
    - 참조자는 별칭임
    - 참조자의 수에는 제한이 없으며, 참조자를 대상으로 참조자를 선언할 수 있음

- 상수 레퍼런스 : [C++](./dia%2001/refConst.cpp)
    - const 선언에 의해서 만들어진 변수를 의미
    - 본래 상수가 아니라, 변수를 상수화시킨 것

## 2일차
- 클래스(class) : [C++](./dia%2002/class.cpp)
    - 클래스를 구성하는 3가지
        1. 접근제어 지시자(private, public, protected)
		    - private : 외부에서 접근 불가
		    - public : 아무나 다 접근 가능	
		    - protected : 상속 관계에서 상속받은 자신만 사용할 수 있음
	    2. 멤버변수 : 속성 -> 외부에서 접근하지 못하도록 private(디폴트값임)영역으로 만들어야함
	    3. 멤버함수(메서드) : 기능 -> 멤버함수를 통해 멤버변수에 접근해야 하기에 public 영역으로 만들어야함
    - 클래스 이름의 첫 글자는 무조건 대문자여야만 함.
    - 클래스는 사용자가 만든 하나의 타입으로 봐야함.
    - 객체(인스턴스)
        - 객체와 인스턴스는 같은 의미이지만 어떤 클래스로 부터 만들어진 객체임을 나타낼 때는 인스턴스 사용

- 생성자(constructor) : [C++](./dia%2002/class_Constructor.cpp)
    - 생성자는 클래스의 이름과 함수의 이름이 동일
    - 반환형이 선언되어 있지 않으며, 실제로 반환하지 않음(즉, 출력 x)
    - 생성자 호출로 객체가 만들어짐
    - 초기화 기능에 사용
    - 프로그래머가 생성자를 작성하면 더 이상 디폴트 생성자는 제공되지 않음
    - 오버로딩 가능
    - 소멸자(destructor)는 기본생성자 앞에 ~ 붙여주면 됨
    - 클래스 및 생성자 예제 : [C++](./dia%2002/class예제.cpp)

- 문자열 처리 총 정리 : [C++](./dia%2002/문자열처리하기.cpp)

- 변수 초기화(intialization) : [C++](./dia%2002/init.cpp)
    - 멤버 초기화(콜론 초기화가 필요한 경우) : [C++](./dia%2002/init_멤버초기화.cpp)
        1. 상수 멤버 변수(const)
        2. 레퍼런스 멤버 변수(참조자)
        3. 객체를 멤버로 가지는 경우

- 2일차 정리 예제 : [C++](./dia%2002/2일차%20최종%20예제.cpp)
